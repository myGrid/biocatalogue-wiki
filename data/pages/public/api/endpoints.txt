===== GET /search?q={query} =====

Keyword/query search to retrieve relevant ''Service'', ''SoapOperation'', ''ServiceProvider'', ''User'' and ''Registry'' resources.

<html>
<div class="box_important">
<b>Note:</b> the following characters have special meaning when used in a search query (used to <i>pattern match</i> in the search engine): \, +, -, !, (, ), :, ^, ], {, }, ~, *, ?

<p>
To search for these characters literally, warp your query in double quotes. 
</p>

<p>
For example, to search for a service that has the WSDL URL "http://ws.adaptivedisclosure.org/axis/services/SearcherWS?wsdl" (which has a question mark in it):
</p>

<a href="http://www.biocatalogue.org/search.xml?q=%22http%3a%2f%2fws.adaptivedisclosure.org%2faxis%2fservices%2fSearcherWS%3fwsdl%22" target="_blank">http://www.biocatalogue.org/search.xml?q=%22http%3a%2f%2fws.adaptivedisclosure.org%2faxis%2fservices%2fSearcherWS%3fwsdl%22</a>

</div>
</html>

====Example====
**URL:** http://www.biocatalogue.org/search.xml?q=dbfetch

**Output Snippet (XML):**
<code xml>
<search xlink:href="http://www.biocatalogue.org/search?q=dbfetch" xsi:schemaLocation="http://www.biocatalogue.org/2009/xml/rest http://www.biocatalogue.org/2009/xml/rest/schema-v1.xsd" resourceType="Search">  <parameters>

    <scope urlValue="all" urlKey="scope">All</scope>
    <page urlKey="page">1</page>

  </parameters>
  <statistics>

    <scopedResults scope="Soap Operations">22</scopedResults>
    ...
  </statistics>
  <results>

===== GET /lookup =====

A special API endpoint to find one particular thing based on some criteria.  This endpoint is avaialable in HTML, XML, and JSON representations.

Currently used to retrieve:

^ Resource ^ Required URL Query Parameters ^
| ''SoapService'' | wsdl_location |
| ''SoapOperation'' | wsdl_location, operation_name |
| ''SoapInput'' | wsdl_location, operation_name, input_name |
| ''SoapOutput'' | wsdl_location, operation_name, output_name |

<html>
<div class="box_important">
<b>Note</b>: at the moment this assumes that there is one and only one result that can be redirected to (via an HTTP 302). However, this may not always be the case and so a future consideration will be to potentially use HTTP 300 redirects to specify that multiple items have been found.
</div>
</html>

Output Formats: depends on the resource that you are redirected to.

**Example URL:** Get a specific ''SoapOperation'' - http://www.biocatalogue.org/lookup?wsdl_location=http://ws.adaptivedisclosure.org/axis/services/IndexWS?wsdl&operation_name=streamContent

** Redirects to:** http://www.biocatalogue.org/soap_operations/15176

===== GET /services =====

''Services'' index/collection resource.

A ''Service'' is the main resource type in the system, and is the container for all things to do with a particular service registered in the BioCatalogue. This includes variants (''SoapService'' and/or ''RestService''), ''ServiceDeployments'', ''ServiceTests'' and so on.

====Example====
**URL:** http://www.biocatalogue.org/services.xml

**Output Snippet (XML):**

<code xml>
<?xml version="1.0" encoding="UTF-8"?>
<services resourceType="Services" xlink:href="http://www.biocatalogue.org/services" ...>
  <parameters>
    <filters>
    </filters>
    <query urlKey="q"></query>
    <sortBy urlKey="sort_by" urlValue="created">Created at date</sortBy>
    <sortOrder urlKey="sort_order" urlValue="desc">Descending</sortOrder>
    <page urlKey="page">1</page>
    <pageSize urlKey="per_page">10</pageSize>
  </parameters>
  <statistics>
    <pages>163</pages>
    <results>1627</results>
    <total>1627</total>
  </statistics>
  <results>
    <service resourceType="Service" xlink:href="http://www.biocatalogue.org/services/2623" xlink:title="Service - G-language SOAP Service">
      <dc:title>Service - G-language SOAP Service</dc:title>
      <name>G-language SOAP Service</name>
      ...    
    </service>
  </results>
  <related>
    <next resourceType="Services" xlink:href="http://www.biocatalogue.org/services?page=2" xlink:title="Next page of results"/>
    <filters resourceType="Filters" xlink:href="http://www.biocatalogue.org/services/filters" xlink:title="Filters for the services index"/>
    ...  
  </related>
</services>
</code>

**URL:** http://www.biocatalogue.org/services.json

**Output Snippet (JSON):**

===== GET /services/filters =====

Filters that can be applied to the ''Services'' index. These include filters for categories, tags, countries, submitters and service providers.

^ Key ^ Filters By ^
| ''cat'' | Service Categories |
| ''t'' | Service Types |
| ''p'' | Service Providers |
| ''su'' | Submitting Users |
| ''sr'' | Submitting Registries |
| ''tag'' | Tags |
| ''tag_s'' | Tags (on Services) |
| ''tag_ops'' | Tags (on Operations) |
| ''tag_rms'' | Tags (on REST Methods) |
| ''tag_ins'' | Tags (on Inputs) |
| ''tag_outs'' | Tags (on Outputs) |
| ''c'' | Country (Location) |

====Example====
**URL:** http://www.biocatalogue.org/services/filters.xml

**Output Snippet (XML):**
<code xml>
<?xml version="1.0" encoding="UTF-8"?>
<filters resourceType="Filters" xlink:href="http://www.biocatalogue.org/Services/filters" ...>
  <group name="Service Categories">
    <type name="Service Categories" description="" urlKey="cat">
      <filter resourceType="Services" name="Sequence Analysis" ...>
        ...
      </filter>
    </type>
  </group>
</filters>
</code>

**URL:** http://www.biocatalogue.org/services/filters.json

===== POST /soap_services =====

__**NOT YET LIVE**__

Submit a SOAP service to the BioCatalogue.

See the examples below on how to structure the request.

**Note**: This action requires authentication using your account credentials. 

Currently only HTTP Basic authentication is supported. For more information on how to use HTTP Basic authentication, please see the following link: [[http://www.w3.org/Protocols/HTTP/1.0/draft-ietf-http-spec.html#BasicAA|HTTP Basic info]]

==== Description of Attributes ====
The following briefly describes each attribute present in the example below.

**Note:** The ability to update service metadata is currently not yet implemented in the API, however __annotations__ can be added at a later time by any user.

^ Attribute      ^ Description      ^ Required? ^
| wsdl_location  | The URL to the WSDL document that describes this service.    | Yes       |
| annotations    | A set of additional metadata to be attached to the service. The different classes of annotation are as follows... | No |
| - alternative_names    | A list of alternative names for the service. | No |
| - categories    | A list of URLs to categories relevant to this service. A list of categories can be found here: http://www.biocatalogue.org/categories.xml | No |
| - description    | A textual description of what the service does. | No |
| - documentation_url    | A URL to some detailed documentation of the service. | No |
| - tags    | A list of text tags relevant to the service. | No |




====Example====
**URL:** http://www.biocatalogue.org/soap_services.xml

**Submitting a SOAP service (XML):**
<code xml>
<soap_service>
    <wsdl_location>http://www.example.com/webservice.wsdl</wsdl_location>
    <annotations>
        <documentation_url>http://www.example.com/docs.html</documentation_url>
        <alternative_names type="array">
            <alternative_name>alternate name for this service</alternative_name>
            <alternative_name>yet another name for this service</alternative_name>
        </alternative_names>
        <tags type="array">
            <tag>example</tag>
            <tag>test</tag>
            <tag>delete me</tag>
        </tags>
        <description>Put a description of the web service here</description>
        <categories type="array">
            <category>http://www.biocatalogue.org/categories/1</category>
            <category>http://www.biocatalogue.org/categories/6</category>
            <category>http://www.biocatalogue.org/categories/17</category>
        </categories>
    </annotations>
</soap_service>
</code>


**Submitting a SOAP service (JSON):**
<code javascript>
{
  "soap_service" : {
    "wsdl_location" : "http://www.example.com/webservice.wsdl",
    "annotations" : {
      "documentation_url" : "http://www.example.com/docs.html",	
      "alternative_names" : ["alternate name for this service", "yet another name for web service"],
      "tags" : ["example", "test", "delete me"],
      "description" : "Put a description of the web service here",
      "categories" : ["http://www.biocatalogue.org/categories/1", "http://www.biocatalogue.org/categories/6", "http://www.biocatalogue.org/categories/17"]
    }
  }
}
</code>

===== POST /rest_services =====

__**NOT YET LIVE**__

Submit a REST service to the BioCatalogue.

See the examples below on how to structure the request.

**Note**: This action requires authentication using your account credentials. 

Currently only HTTP Basic authentication is supported. For more information on how to use HTTP Basic authentication, please see the following link: [[http://www.w3.org/Protocols/HTTP/1.0/draft-ietf-http-spec.html#BasicAA|HTTP Basic info]]

==== Description of Attributes ====
The following briefly describes each attribute present in the example below.

**Note:** The ability to update service metadata is currently not yet implemented in the API, however __annotations__ can be added at a later time by any user.

^ Attribute      ^ Description      ^ Required? ^
| name           | The name of the service.     | Yes       |
| endpoint       | The "base"/"root" endpoint of the service.     | Yes       |
| rest_resources | A list of endpoints relevant to this service. See below. | No |
| annotations    | A set of additional metadata to be attached to the service. The different classes of annotation are as follows... | No |
| - alternative_names    | A list of alternative names for the service. | No |
| - categories    | A list of URLs to categories relevant to this service. A list of categories can be found here: http://www.biocatalogue.org/categories.xml | No |
| - description    | A textual description of what the service does. | No |
| - documentation_url    | A URL to some detailed documentation of the service. | No |
| - tags    | A list of text tags relevant to the service. | No |


=== REST Resources === 
Each endpoint should consist of a string containing: 
  * the relevant HTTP verb (GET, POST, PUT, DELETE, etc.)
  * a space
  * the URL of the endpoint, including the base endpoint, with any URL parameters marked with an identifier surrounded by curly braces - {}

For example, the endpoint:
  GET http://www.example.com/search.php?query={query_string}
...would accept an HTTP GET request to the following example URL:
  http://www.example.com/search.php?query=higgs+boson




====Example====
**URL:** http://www.biocatalogue.org/rest_services.xml

**Submitting a REST service (XML):**
<code xml>
<rest_service>
    <name>Example Service</name>
    <endpoint>http://www.example.com</endpoint>
    <rest_resources type="array">
        <rest_resource>POST http://www.example.com/examples</rest_resource>
        <rest_resource>GET http://www.example.com/examples</rest_resource>
        <rest_resource>GET http://www.example.com/examples/{id}</rest_resource>
        <rest_resource>PUT http://www.example.com/examples/{id}</rest_resource>
        <rest_resource>DELETE http://www.example.com/examples/{id}</rest_resource>
        <rest_resource>GET http://www.example.com/search.php?query={query_string}</rest_resource>
    </rest_resources>    
    <annotations>
        <documentation_url>http://www.example.com/docs.html</documentation_url>
        <alternative_names type="array">
            <alternative_name>alternate name for this service</alternative_name>
            <alternative_name>yet another name for this service</alternative_name>
        </alternative_names>
        <tags type="array">
            <tag>example</tag>
            <tag>test</tag>
            <tag>delete me</tag>
        </tags>
        <description>Put a description of the web service here</description>
        <categories type="array">
            <category>http://www.biocatalogue.org/categories/1</category>
            <category>http://www.biocatalogue.org/categories/6</category>
            <category>http://www.biocatalogue.org/categories/17</category>
        </categories>
    </annotations>
</rest_service>
</code>

**Submitting a REST service (JSON):**
<code javascript>
{
  "rest_service" : {
    "name" : "Example REST service",
